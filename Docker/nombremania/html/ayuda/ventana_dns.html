¿Qué es una DNS?
DNS es la tecnología que enlaza el texto del nombre de un dominio y su número de dirección IP, lo cual es necesario para localizar el servidor del dominio en la red. Al escribir un dominio en tu navegador (www.dominio.com) la DNS convierte esta dirección en la IP que corresponde al servidor de nombre registrado con el dominio (123.123.123.1). Vea Como funciona una DNS para más información

¿Qué es un registro A?
Un registro A, también denominado registro "Address", enlaza un nombre de dominio con una dirección IP. Si hay un servidor final operativo que esté configurado para conducir el tráfico del dominio en esa dirección, el dominio se resolvera de forma inmediata al escribir el nombre. Es el tipo de entrada habitual al configurar un servidor web para hospedar un dominio. Cada entrada de dominio (del tipo loquequieras.tudominio.com) normalmente solo cuenta con una IP asociada, salvo en el caso de querer usar “Round Robin” (balanceo de servidores) y contar con más de un servidor operativo.
¿Qué es un registro MX?
Los registros "MX" (Mail eXchanger) se usan para especificar qué servidor de Internet está configurado para conducir los emails a tu dominio. Al igual que las entradas tipo “A”, si hay un servidor correctamente configurado en esa IP todo el trafico de correo será enviado a ese servidor. Si este no estuviera operativo o bien configurado, TODO el correo sería devuelto a su origen.
¿Qué es el "ranking" de servidores MX?
El servicio PRO de DNS de NombreMania permite incluir más de un servidor de correo de forma encadenada. Actuando el segundo como servidor “backup” en caso de estar el primero inoperativo, y así sucesivamente.
¿Qué es un registro CNAME?
NO USES CNAME SI NO ENTIENDES PERFECTAMENTE PARA QUE SIRVE.
Debes tener mucho cuidado al usar esta opción que se recomienda solo para usuarios avanzados. Las entradas CNAMEs fueron creadas para redirigir todo el tráfico de una DNS a otro dominio. Por esta razón supone una violación al protocolo DNS tener un CNAME y cualquier otro registro DNS para el mismo dominio.
En general lo que buscas puede resolverse con registros (A) y/o uso de WebForward® o MailForward®.
¿Como funciona el balanceo de servidores (Round Robin)?
Round-robin DNS es una técnica que permite que múltiples servers atiendan un mismo dominio, para que exista un balanceo de carga entre ellos. Esta técnica sirve especialmente para los servidores web pero puede ser usada para otro tipo de servidores.
Lo único que debe hacer es incluir varias IPs para el mismo dominio. Nuestros servidores balancearán la carga entre estos IPs en forma automática.

¿Por qué no puedo tener el mismo nombre de dominio en multiples tipos de registros?
Por qué un WebForward® necesita un registro A especial para funcionar correctamente. Además, CNAMEs son entre sí exclusivos de registros A, por lo que un nombre de dominio no puede ser ambos, una dirección IP y un alias simultáneamente.
¿Por qué no puedo borrar el registro A del nombre raíz de mi dominio?
 Todos los dominios deben tener un registro A en la raíz del nombre del dominio. Omitiendo este registro A puedes impedir que algún servidor de correo envíe tu correo correctamente. Un Webparking® o WebForward® en la raíz de tu dominio, será suficiente, por qué en ambos casos se creará un registro A “oculto” que apuntará a nuestro servidor.
¿Cuando se aplican los cambios?
Todos los cambios individuales de DNS se ajustarán inmediatamente y normalmente son aplicables en menos de un minuto. El problema es que los ISP (proveedores de acceso a internet) en algunos casos tienen guardadas las configuraciones de dns usadas (cache) y estos cambios pueden demorarse (en particular a usted si ha accedido antes) hasta una hora.
